## 寒假大礼包-省点心

- 项目介绍:省点心是一款管理物流的网站

### 功能模块

- 发货单模块

```
功能逻辑：
发货单这个模块是查看物流状态信息的,其中又有4个状态发货、收货、评价、查看评论。
获取这4个状态则需要通过对接口传参才可以获取对应状态。
操作这些状态并实现对应功能需要登录，即需要获取token值并带入请求头中才可以操作这些物流状态。
登录后我直接在axios的请求拦截器中把所有接口的请求头中添加了token数据，如果非法登录则无法获取到数据
总体逻辑其实都是调接口然后传对应参数就行了，如果参数传的不正确会报500错误，一般通过判断状态码来提示用户操作是否成功。

遇到的问题：
通过keep-alian设置页面缓存时，我使用include这个方法排除不缓存的组件，但对动态路由不起效果，最后通过在路由的meta信息中加入判断条件阻止了动态路由的缓存
```

- 消息中心+登录+个人中心

```
消息中心:
消息中心和发货单的其实差不多,都是通过对应的参数渲染对应的数据,标签栏以及分页等功能因为有许多共用的地方就将其封装成了一个组件,便于维护与快速开发.
登录:
登录功能就是向后台发送账号密码,来验证用户身份,如果账号密码错误则无法登录,登录后用localStroage存储token数据,并在axios请求拦截中为所有后续的请求添加请求头带入token数据,以便调用接下来的其他接口,简化操作。
个人中心:
用户登录后就可以通过token数据获取到对应的个人中心数据,退出登录后将移除localStorage存储的token数据,用户需要重新登录才可获取到信息

遇到的问题:
在做分页功能中,子组件修改父组件传过来的数据会报错,警告说不能直接修改参数。
解决办法:通过子传父的方式通知父组件修改,或者在子组件标签中给需要修改的属性前面加上" .sync "即可解决报错问题
```
